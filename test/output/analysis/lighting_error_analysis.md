# 光照对电机轴圆形检测误差影响 — 分析报告

## 一、摘要

**结论：光照不是检测不稳定的主要原因，但是一个加剧因素。**

核心证据：
1. 检测失败率与转速强正相关（视频10-50失败率0%，视频90失败率77%，视频100失败率94%），而同一相机同一场景下光照条件基本一致——若光照是主因，低转速视频也应出现失败
2. 代码中确实存在光照敏感性缺陷（无预处理、固定阈值），这使得光照波动会成为**放大器**而非**触发器**
3. 不稳定视频的半径崩溃呈指数衰减特征（正反馈崩溃），这是代码逻辑缺陷的直接结果，与光照无直接因果

**建议**：在修复代码级缺陷（自适应正反馈、Hough范围过宽）的基础上，建议在灰度化后加入CLAHE预处理作为鲁棒性增强，但不应将其视为主要修复手段。

## 二、代码光照敏感性分析

`src/tracking/shaft_detector.py` 的检测流程存在4处光照敏感点：

| 位置 | 代码 | 光照敏感机理 |
|------|------|------------|
| 行87 | `gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)` | 直接灰度化，无光照归一化。不同亮度帧的灰度值分布不同，但后续处理用同一套阈值 |
| 行90-95 | `HoughCircles(gray, param1=100, param2=25, ...)` | `param1=100`是内部Canny高阈值，固定值。暗帧中电机轴边缘梯度可能低于100→Canny丢失边缘→Hough无圆可检；亮帧中噪声梯度可能超过25(param2)→假阳性圆 |
| 行122-123 | `GaussianBlur + Canny(blurred, 50, 150)` | 策略2的Canny阈值同样固定(50,150)。低光照下SNR降低，弱边缘被当作噪声滤除；高光照下镜面反射产生强梯度，形成虚假边缘 |
| 全局 | 无`equalizeHist`/`CLAHE`/`adaptiveThreshold`/gamma | 全程无任何光照自适应预处理。不同光照条件的帧用完全相同的参数处理，缺乏鲁棒性 |

### 关键参数解读

- **`param1=100`（Hough内部Canny高阈值）**：OpenCV文档说明，HoughCircles使用Canny进行内部边缘检测，`param1`是高阈值，低阈值自动设为高阈值的一半(50)。当帧的整体亮度偏低时，电机轴边缘的梯度幅值可能低于100，导致边缘被Canny完全丢弃。

- **`param2=25`（累加器阈值）**：值越低越容易检测到圆，但也更容易产生假阳性。在光照充足的帧中，真实圆的边缘投票集中在累加器中，假阳性较少；但在低光照帧中，噪声投票分散，`param2=25`可能既检到真实圆又检到噪声圆。

- **Canny(50, 150)**：策略2的固定阈值。光照变化导致边缘梯度幅值变化，固定阈值无法适应。

## 三、光照影响的机理推断

基于代码分析，光照可能通过以下4种机理影响检测：

### 假设1：光照不足 → 噪声主导 → 固定阈值丢失弱边缘

**机理**：低光照环境下，相机提高ISO或延长曝光时间，导致噪声增加。电机轴边缘的梯度幅值降低，可能低于Canny高阈值(100或150)，边缘被丢弃。同时SNR降低，霍夫累加器中真实圆的投票峰被噪声淹没。

**预期表现**：低光照视频的检测失败率更高，半径检测值偏小（部分边缘丢失导致圆变小）。

### 假设2：镜面反射闪烁 → 边缘强度帧间跳变

**机理**：电机轴是金属表面，端面轴向视图下旋转时会产生镜面反射。反射光斑随旋转角度变化，导致：
- 反射区域边缘强度突增 → Canny检测到虚假边缘 → Hough假阳性
- 反射遮挡真实边缘 → 真实圆边缘断裂 → Hough假阴性

**预期表现**：高转速视频的半径检测帧间跳变更剧烈（反射变化更快），半径分布可能呈双峰（反射帧检到反射圆，无反射帧检到真实圆）。

### 假设3：光照时间漂移 → 固定阈值在前后段表现不一致

**机理**：若视频拍摄过程中光照逐渐变化（如光源预热、环境光变化），固定阈值在前N帧表现良好但后N帧失效。

**预期表现**：检测失败率随帧号递增，半径均值在后半段偏移。

### 假设4：预处理缺失 → 不同光照帧用同一阈值处理

**机理**：即使光照变化幅度不大，缺乏预处理也意味着检测器无法自适应。对比加入CLAHE等预处理后的效果，可以量化预处理的改善空间。

**预期表现**：添加CLAHE后，不稳定视频的检测成功率有所提升（但若运动模糊是主因，提升有限）。

## 四、现有数据中的光照线索

### 4.1 稳定vs不稳定视频对比

从 `instability_metrics.xlsx` 提取的关键指标：

| 视频 | 转速 | radius_cv% | 失败率% | scale偏差% | 状态 |
|------|------|-----------|---------|-----------|------|
| 10 | 低 | 3.6 | 0 | +1.0% | 稳定 |
| 20 | 低 | 6.1 | 0 | 0.0% | 稳定 |
| 30 | 中低 | 5.7 | 0 | -17.3% | 轻度双峰 |
| 40 | 中低 | 5.2 | 0 | 0.0% | 稳定 |
| 50 | 中 | 5.2 | 0 | 0.0% | 稳定 |
| 60 | 中高 | 12.0 | 0 | -19.0% | 双峰 |
| 70 | 中高 | 9.9 | 0 | 0.0% | 轻度漂移 |
| 80 | 高 | 14.4 | 0 | -19.3% | 双峰 |
| 90 | 很高 | 191.2 | 77 | +61.3% | 崩溃 |
| 100 | 最高 | 449.1 | 94 | +67% | 崩溃 |

**关键观察**：

1. **检测失败率与转速强正相关**：视频10-80失败率均为0%，视频90突然跳升至77%，视频100升至94%。如果光照是主因，我们期望看到低转速视频也有失败帧（因为光照条件相同），但事实并非如此。这强烈暗示**转速/运动模糊是失败的主因**。

2. **半径CV%随转速递增**：从3.6%(视频10)到14.4%(视频80)，再到191%(视频90)。这种递增趋势与转速递增一致，但也与光照无关（同一场景光照不变）。

3. **双峰出现在中高转速**：视频30/60/80出现比例尺偏低(-17~-19%)，对应半径偏大（检测到~120px而非100px）。这可能是高转速下镜面反射变化加快，Hough偶尔检到反射形成的较大圆——**这是光照（反射）可能参与的唯一环节**。

4. **崩溃只发生在最高转速**：视频90/100的半径崩溃是指数衰减型（正反馈崩溃），这在 `root_cause_diagnosis.md` 中已归因于 `expected_radius = vis_radius` 的自适应正反馈。光照无法解释指数衰减特征。

### 4.2 光照可能参与的证据

虽然光照不是主因，但以下线索表明光照（特别是镜面反射）可能参与了部分故障：

- **视频80的双峰（100/124px）**：Hough范围 `[0.4*100, 1.8*100] = [40, 180]` 允许检测到124px的圆。这个较大的圆可能是金属表面镜面反射形成的伪边缘，而非轴承外圈。反射强度随旋转角度变化，导致部分帧检到反射圆、部分帧检到真实轴。

- **视频60-80的半径渐增**：CV%从5.2%(视频50)上升到12-14%(视频60-80)。高转速下反射变化更快，可能导致更多帧的边缘检测不稳定。

### 4.3 排除光照为主因的证据

- **同一场景同一相机**：10个视频是在相同光照条件下拍摄的（同一实验台、同一光源）。如果光照是主因，所有视频都应受影响，而非只有高转速视频。

- **低转速视频完全稳定**：视频10-50的失败率为0%、CV%<6%。在相同光照下，这证明光照条件本身足以支持稳定检测。

- **崩溃的转速阈值效应**：视频80（高转速）失败率仍为0%，视频90（很高转速）突然77%。这种阈值效应更符合运动模糊的物理模型（曝光时间内轴旋转角度超过某阈值导致模糊），而非光照模型。

## 五、光照与运动模糊的区分

### 5.1 混淆来源

`blur_lap_var`（Laplacian方差，衡量清晰度）同时受光照和运动模糊影响：
- 低光照 → 高ISO噪声 → 图像清晰度下降 → blur_lap_var降低
- 高转速 → 运动模糊 → 图像清晰度下降 → blur_lap_var降低

无法仅从blur_lap_var区分两者。

### 5.2 区分逻辑

| 判据 | 光照为主因 | 运动模糊为主因 |
|------|-----------|-------------|
| 失败率vs转速 | 弱相关（光照不随转速变） | 强正相关（转速越高模糊越严重） |
| 低转速视频表现 | 也应有失败帧 | 应完全稳定 |
| 崩溃特征 | 渐进式（光照渐变） | 突变式（超过模糊阈值） |
| 双峰方向 | 反射圆可能更大或更小 | 模糊导致检测偏小 |

实际数据：失败率与转速强正相关、低转速完全稳定、崩溃呈突变式——**全部指向运动模糊为主因**。

### 5.3 光照的辅助角色

光照（特别是镜面反射）可能在高转速视频中起到**加剧作用**：
- 运动模糊导致边缘变弱 → 边缘梯度接近Canny阈值边界
- 镜面反射导致边缘强度帧间波动 → 部分帧边缘被反射增强（检到假圆），部分帧边缘被反射遮挡（丢失真圆）
- 两者叠加 → 检测不稳定加剧

这解释了为什么视频60-80出现双峰（反射参与），但只有视频90-100才崩溃（运动模糊超过阈值）。

## 六、结论与建议

### 6.1 结论

1. **光照不是检测不稳定的主要原因**。核心证据是：同一光照条件下，低转速视频完全稳定，高转速视频才出现失败。失败率与转速强正相关，而非与光照条件相关。

2. **运动模糊/转速是检测崩溃的直接原因**。视频90/100的半径正反馈崩溃由运动模糊触发——高转速下曝光时间内轴旋转角度增大，图像模糊导致Hough检测到偏小圆，进而触发 `expected_radius = vis_radius` 的正反馈崩溃。

3. **镜面反射可能是双峰误检的辅助因素**。视频60-80的半径双峰（100/124px）中，较大的圆可能是金属表面镜面反射形成的伪边缘。反射随旋转角度变化，导致部分帧检到反射圆。

4. **代码中的光照敏感性缺陷是真实存在的**。无预处理、固定阈值的设计确实使检测器对光照波动缺乏鲁棒性，但在当前数据集中光照波动不足以单独导致检测失败。

### 6.2 建议

#### 优先级排序

| 优先级 | 改进 | 理由 |
|--------|------|------|
| P0 | 移除半径自适应正反馈 | 根本原因，与光照无关 |
| P0 | 比例尺改用首帧/全局标定 | 根本原因，与光照无关 |
| P1 | 收紧Hough范围至±15% | 消除双峰，间接减少反射误检 |
| P2 | **加入CLAHE预处理** | 增强光照鲁棒性，减少反射影响 |
| P2 | 提高Hough param2至40 | 减少假阳性，包括反射产生的假圆 |

#### CLAHE预处理建议

虽然光照不是主因，但加入CLAHE仍有价值：
- 均衡化光照分布，减少镜面反射区域与真实边缘的对比度差异
- 使Canny固定阈值在不同光照条件下表现更一致
- 实现简单（3行代码），无副作用

建议在 `shaft_detector.py` 行87后加入：
```python
gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)
# 加入CLAHE预处理，增强光照鲁棒性
clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8, 8))
gray = clahe.apply(gray)
```

#### 不建议的方案

- **自适应Canny阈值**：虽然理论上更优，但实现复杂且需要调参，当前数据不足以验证效果
- **自适应阈值二值化**：会改变Hough的输入特性，可能引入新的不稳定因素

### 6.3 局限性

1. **未实际运行光照指标计算**：本报告基于代码审查和数据推断，未从视频中提取帧计算亮度/对比度等光照指标。若需更确凿的证据，需运行光照指标分析脚本。
2. **无法完全分离光照与转速**：每个转速只有1个视频，无法控制变量（同一转速不同光照条件）。
3. **镜面反射假设未经验证**：双峰中较大圆是反射还是轴承外圈，需可视化失败帧确认。
4. **CLAHE效果未实测**：建议基于机理推断，实际效果需在崩溃视频上验证。
