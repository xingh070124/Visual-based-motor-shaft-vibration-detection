# 电机轴圆形检测不稳定性 — 根因诊断报告

## 一、诊断总览

通过对10个视频输出数据的量化分析和源代码审查，确认检测不稳定由**8个代码级缺陷**引起，
其中**2个致命**、**1个高危**、**4个中等**、**1个低风险**。

## 四、三个故障模式详解

### 模式A — 半径正反馈崩溃（致命）

**触发条件**：高转速视频（90/100），电机轴表面运动模糊增加，Hough检测到偏小圆。

**崩溃过程**（以视频90为例）：
1. 前50帧：运动模糊导致检测半径从100px逐渐降至66px
2. `expected_radius` 跟随降至66px → Hough范围从[40,180]缩至[26,119]
3. 第200帧：expected降至64px → 范围[26,115] → 更难检到100px的圆
4. 第350帧：expected降至36px → 范围[14,65] → 完全无法检到100px的轴
5. 第400帧后：半径=0（检测失败） → expected冻结在最后有效值
6. 前50帧中位数R=62px → scale=12000/(2*62)=96.8μm/px（理论60的1.6倍）

**关键代码**（`batch_analysis.py:478-480`）：
```python
# 自适应先验：检测成功则更新
if vis_radius > 0:
    expected_radius = vis_radius  # ← 正反馈根源
```

### 模式B — 半径双峰漂移（高）

**触发条件**：视频30、60、80，电机轴附近有轴承外圈等更大圆形特征。

**误检过程**：
1. expected~100px → Hough范围 [40, 180]
2. 部分帧的Hough检测到轴承外圈（~120-124px）而非电机轴（~100px）
3. `expected_radius` 自适应更新至更大值 → 范围进一步扩大
4. 两种检测结果交替出现 → 半径呈双峰分布
5. 前50帧中位数R~121-124px → scale=12000/(2*121~124)=48-50μm/px（理论60的0.81倍）

**实际影响**：
| 视频 | R_mean(px) | R_median(px) | scale(μm/px) | 偏差 |
|------|-----------|-------------|-------------|------|
| 30 | 101.1 | ~121 | 49.6 | -17.3% |
| 60 | 105.2 | ~123 | 48.6 | -19.0% |
| 80 | 120.6 | ~124 | 48.4 | -19.3% |

**关键代码**（`shaft_detector.py:93-94`）：
```python
minRadius=int(expected_radius_pixels * 0.4),   # 40px — 太低
maxRadius=int(expected_radius_pixels * 1.8)    # 180px — 太高
```

### 模式C — 比例尺腐蚀连锁（致命，模式A/B的后果）

**传导路径**：
```
半径检测异常 → R_median偏移 → scale = D/(2*R_median) 偏移 → 振幅 = scale * 像素位移 失真
```

**量化影响**：
| 视频 | R_median | scale (μm/px) | 偏差 | X振幅原始 | X振幅修正 | 放大倍数 |
|------|---------|--------------|------|----------|----------|---------|
| 90 | ~62 | 96.8 | +61% | 1.88mm | 1.17mm | 1.61x |
| 100 | ~60 | 100.0 | +67% | 1.79mm | 1.08mm | 1.67x |
| 30 | ~121 | 49.6 | -17% | 0.54mm | 0.66mm | 0.83x |
| 60 | ~123 | 48.6 | -19% | 0.49mm | 0.60mm | 0.81x |
| 80 | ~124 | 48.4 | -19% | 0.61mm | 0.76mm | 0.81x |

**结论**：视频90/100的高振幅（1.8+mm）是**测量伪影**，修正后实际振幅为1.1mm左右，与低转速视频相当。
视频30/60/80的振幅被**低估**约19%，修正后应增大约20%。

## 五、振动数据架构问题

`batch_analysis.py:465-467` 中，振动位移使用STARK bbox中心而非圆检测中心：
```python
track_cx = pred_box[0] + pred_box[2] / 2  # bbox几何中心
track_cy = pred_box[1] + pred_box[3] / 2
```

这意味着：
- **坐标抖动**完全来自STARK跟踪器（亚像素预测抖动），与圆检测无关
- **振幅失真**完全来自比例尺腐蚀（半径检测异常），与坐标抖动无关
- 两个问题独立，需分别解决

图8验证了这一点：视频90的Pixel_X std与稳定视频相近，但Cam_X std被放大4.7倍，
完全由比例尺膨胀解释。

## 六、总结

| 问题类别 | 根本原因 | 影响 | 修复优先级 |
|---------|---------|------|-----------|
| 检测崩溃 | 自适应半径正反馈 | 视频90/100完全失效（77%/94%帧失败） | P0（立即） |
| 振幅高估 | 比例尺被崩溃半径膨胀 | 视频90/100振幅放大1.6~1.7倍 | P0（立即） |
| 振幅低估 | 比例尺被双峰半径压缩 | 视频30/60/80振幅低估约19% | P1（高） |
| 误检大圆 | Hough范围过宽 | 视频30/60/80检测到轴承外圈 | P1（高） |
| 坐标噪声 | 无时域滤波 | 全部视频高频抖动 | P2（中） |
| 漂移伪影 | 无去趋势 | 振幅偏大 | P2（中） |
